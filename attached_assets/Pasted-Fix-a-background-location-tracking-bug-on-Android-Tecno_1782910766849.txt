Fix a background location tracking bug on Android (Tecno/HiOS devices, Android 16).

to the TaskManager callback (LOCATION_TASK_NAME) after the app is backgrounded 
for ~20 minutes — even though the foreground service notification stays visible 
and Doze battery optimization is confirmed whitelisted. The expo-background-task 
watchdog (WATCHDOG_TASK_NAME) also never fires — diagnostics screen shows 
"Background task last ran: never" even after 23+ minutes, when it should run 
every ~15 minutes.

   intent with data "package:com.kssolar.app". If that throws, fall back to 
   launching IntentLauncher.ActivityAction.APPLICATION_DETAILS_SETTINGS for the 
   same package. Install expo-intent-launcher via `npx expo install expo-intent-launcher`

   background location permission is confirmed granted (bgStatus === "granted"),
